# 模糊定位实现说明（给新同学看的版本）

> 重要程度用星级标记：★★★★★ 最高，★☆☆☆☆ 最低。  
> 建议第一次看按顺序从上往下读一遍。

---

## 0. 整体总览（★★★★★）

这套功能解决的问题是：

> 用户输入一段中文地理描述 →  
> deepseek 把它转成「空间三元组」 →  
> geolocus 在「上下文」里根据这些三元组做空间推理 →  
> 前端把结果画成：模糊区域 + 概率栅格 + 代表点。

相关关键文件（建议先把这些文件在 IDE 里都打开）：

- 三元组示例和 deepseek 提示词：`package/client/src/util/deepseek.util.ts`
- geolocus 上下文与计算封装：`package/client/src/util/geolocus.util.ts`
- 聊天流程与地图调用：`package/client/src/page/chat/Chat.tsx`
- 地名解析插件：`package/client/src/util/place.plugin.ts`

## 1. 核心概念

### 1.1 `UserGeolocusTriple`：空间三元组（★★★★★）

**作用：**  
用一个结构化对象描述一句话里的「空间约束」。这是 geolocus 接收的**数据格式**。

**类型定义位置：**

- `package/client/src/util/deepseek.util.ts:10-111`（写在 `prompt` 字符串里）

简化后的结构（原始定义在字符串里）：

```ts
interface UserGeolocusTriple {
  // 描述一组“参照物 + 关系 + 目标”
  tupleList: {
    // 参照物数组：可以是简单对象，也可以嵌套三元组
    originList: (
      | {
          name: string
          type?:
            | 'Point'
            | 'LineString'
            | 'Polygon'
            | 'MultiPoint'
            | 'MultiLineString'
            | 'MultiPolygon'
          coord?:
            | [number, number]
            | [number, number][]
            | [number, number][][]
            | [number, number][][][]
        }
      | UserGeolocusTriple
    )[]
    // 目标和这一堆参照物的空间关系
    relation?: {
      topology?: 'disjoint' | 'contain' | 'within' | 'intersect' | 'along'
      direction?:
        | 'N'
        | 'NE'
        | 'E'
        | 'SE'
        | 'S'
        | 'SW'
        | 'W'
        | 'NW'
        | 'F'
        | 'FR'
        | 'R'
        | 'BR'
        | 'B'
        | 'BL'
        | 'L'
        | 'FL'
      distance?: number | [number, number] | 'VN' | 'N' | 'M' | 'F' | 'VF'
      range?: 'inside' | 'outside' | 'both'
    }[]
    // 目标物名称（要推断的位置是谁）
    target: string
  }
}
```

你不需要记住全部字段，只要知道：

- `originList`：参照物列表，里面写的是地名（和可选几何）。  
- `relation`：目标和参照物之间的关系（方向 / 距离 / 拓扑 等）。  
- `target`：目标物名称（“新街口”“目标对象”等）。

### 1.2 geolocus 库 `@geolocus/core`（★★★★★）

**作用：**  
一个「空间推理引擎」：

- 输入：`UserGeolocusTriple[]` + 上下文配置  
- 输出：模糊点 / 线 / 面 + 概率栅格 + 每条约束的中间结果

**引入位置：**

- `package/client/src/util/geolocus.util.ts:2`

```ts
import { Geolocus, geolocus, UserGeolocusTriple } from '@geolocus/core'
```

你在前端主要用到这些 API：

- `geolocus.createContext(options)`：创建 geolocus 上下文。  
- `context.addRole(roleConfig)`：配置角色（方向/距离误差、语义距离映射等）。  
- `context.use('placePlugin', fn)`：挂载地名解析插件。  
- `context.defineRelation(tripleList, 'point' | 'line' | 'polygon')`：导入三元组。  
- `context.computeFuzzyPointObject(targetName)` / `computeFuzzyLineObject` / `computeFuzzyPolygonObject`：做模糊点/线/面推理。  
- `context.toGeoJSON(geoObject)`：把内部几何转成 GeoJSON。

可以把 geolocus 理解为：

> **给它“关系 + 配置”，它负责“在地图上帮你找到最符合这些关系的结果”。**

### 1.3 geolocusContext 上下文（★★★★★）

**作用：**  
一次 geolocus 计算的「世界观 + 规则 + 状态容器」。

**创建与配置位置：**

- `package/client/src/util/geolocus.util.ts:5-46`

```ts
// geolocus.util.ts:5-18
export const initContext = () => {
  const geolocusContext = geolocus.createContext({
    maxDistance: 1000000,
    name: '测试上下文',
    gridSum: 256 * 256,
    region: [
      [-99999999, -99999999],
      [99999999, -99999999],
      [99999999, 99999999],
      [-99999999, 99999999],
      [-99999999, -99999999],
    ],
    gridScale: 1000,
  })

  geolocusContext.addRole({
    name: 'default',
    directionDelta: 20,
    distanceDelta: 0.2,
    orientation: 0,
    timeDistanceMap: new Map([
      ['飞机', 300],
      ['步行', 1.67],
    ]),
    semanticDistanceMap: {
      VN: [0, 100],
      N: [0, 200],
      M: [300, 1000],
      F: [1000, 3000],
      VF: [3000, 20000],
    },
    weight: 1,
    spatialRef: geolocus.createSpatialRefFromEPSG(
      geolocus.generateUUID(),
      '4326',
    ),
  })

  geolocusContext.use('placePlugin', getPlaceDataByName)

  return geolocusContext
}
```

**关键配置含义：**

- `region` / `gridSum` / `gridScale`（决定「计算空间」和「分辨率」）
  - `region`：一个巨大的矩形（近似整个世界），geolocus 只在这块区域里算。  
  - `gridSum: 256 * 256`：把 `region` 均匀切成 256×256 个格子，每格一个概率值。  
  - `gridScale: 1000`：每格大致 1000 的空间尺度（视内部单位而定），影响精度和性能。

- `maxDistance: 1000000`  
  - 限制关系中可接受的最大距离，防止搜索范围无限扩大。

- `addRole({ ... })`：定义角色 `default` 的「解释规则」
  - `directionDelta: 20`：方向允许 ±20° 的误差。  
  - `distanceDelta: 0.2`：距离允许 ±20% 的误差。  
  - `semanticDistanceMap`：把 VN/N/M/F/VF 这类语义距离转换成数值区间。  
  - `timeDistanceMap`：把“飞机/步行”等文字换成速度，用于把时间换算为距离。  
  - `spatialRef`：把坐标系设为 EPSG:4326（经纬度）。

- `use('placePlugin', getPlaceDataByName)`  
  - 注册地名解析插件，后面 geolocus 看到 `originList.name = '羊山公园'` 时，会通过这个插件查出它的几何位置。

> 总结：**上下文负责统一“我们在哪儿算”和“怎么解释三元组里的内容”。**

### 步骤 5：导入三元组并执行模糊点推理（★★★★★）

**目的：**

1. 把步骤 3 得到的 `UserGeolocusTriple[]` 导入 context。  
2. 告诉 geolocus：这是「点」类型问题（目标是一个点）。  
3. 让 geolocus 在 context 中执行模糊点推理，得到：区域 / 栅格 / 代表点。

**代码位置：**

- `package/client/src/util/geolocus.util.ts:49-54`

```ts
// geolocus.util.ts:49-54
export const computePointTest = (geolocusContext: Geolocus, text: string) => {
  const tripleList = JSON.parse(text) as UserGeolocusTriple[]
  geolocusContext.defineRelation(tripleList, 'point')
  const res = geolocusContext.computeFuzzyPointObject(tripleList[0].target!)
  console.log(res)
  return res
}
```

`computePointTest` 一般是在 `Chat.tsx` 中这样被调用（简化）：

```ts
// Chat.tsx:1033-1039
if (geometryType === 'point') {
  // 这里目前还会额外调用 xinjieoku() 做示例渲染
  xinjieoku()
  pointTest(jsonText)  // 内部会 initContext() 然后 computePointTest(...)
}
```

**输入：**

- `geolocusContext`：已经配置好的上下文。  
- `text`：步骤 3 的 `jsonText`（字符串形式的三元组数组）。

**内部做了什么：**

1. `JSON.parse(text)`  
   - 把三元组 JSON 字符串还原为 `UserGeolocusTriple[]`。

2. `geolocusContext.defineRelation(tripleList, 'point')`  
   - 遍历每条三元组：  
     - 用 `placePlugin` 把 `originList.name` 变成带几何的参照物。  
     - 按角色配置，把 `direction/distance/topology` 转成可计算的条件（考虑误差、语义距离等）。  
     - 建立“目标物 ↔ 参照物”的关系网络，存入 context。

3. `geolocusContext.computeFuzzyPointObject(tripleList[0].target!)`  
   - 以第一条三元组的 `target`（比如 `'新街口'`）作为目标名称。  
   - 在 `region` 范围内，以栅格为单位：  
     - 对每个栅格中心点，计算它是否满足所有三元组里的约束。  
     - 形成一个二维“可能性分布”矩阵（`regionPdfGrid`）。  
   - 从这个矩阵中：  
     - 提取高概率区域 → `region`（一个 `GeolocusObject` 的 Polygon）。  
     - 提取一个代表点 → `result`（一个 `GeolocusObject` 的 Point）。

**输出：**

- `res`：geolocus 的结果对象，主要字段：
  - `res.region`: 高概率区域（多边形）。  
  - `res.regionPdfGrid`: 概率栅格（二维数组）。  
  - `res.result`: 代表点。  
  - `res.geoTripleResultList`: 每条三元组对应的中间结果。

### 步骤 6：把 geolocus 结果画到地图上（★★★★☆）

**目的：**

把 `res.region` / `res.regionPdfGrid` / `res.result` 变成 Mapbox 上的三个图层：

- 半透明 Polygon（模糊区域）  
- 覆盖在区域上的 PNG 图片（概率栅格）  
- 红点（代表位置）

**代码位置（以通用点模式 `pointTest` 为例）：**

- 在 `package/client/src/page/chat/Chat.tsx` 中搜索 `pointTest`。

**核心逻辑（简化版）：**

```ts
const region = res.region as GeolocusObject
const pdfGrid = res.regionPdfGrid as GeolocusGrid
const result = res.result as GeolocusObject
const coord = result.getGeometry().getCenter() as Position2

// 1. 画区域 (Polygon)
const polygon84 = toWgs84(geolocusContext.toGeoJSON(region))
addGeoJSONToMap(map, region.getUUID() + 'region', polygon84, 'fill', {
  'fill-outline-color': '#15803d',
  'fill-color': '#4ade80',
  'fill-opacity': 0.5,
})

// 2. 画概率栅格 (PNG)
const pngBlob = generateBlobPng(pdfGrid)
const bbox = convertToWgs84(
  region.getGeometry().getBBox().slice(0, 2) as Position2,
).concat(
  convertToWgs84(region.getGeometry().getBBox().slice(2, 4) as Position2),
)
addImageToMap(map, region.getUUID() + 'regionPdfGird', pngBlob, bbox)

// 3. 画代表点 (Point)
const coord84 = convertToWgs84(coord)
const point = generatePointByCoord(coord84)
addGeoJSONToMap(map, result.getUUID() + 'result', point, 'circle', {
  'circle-color': '#dc2626',
  'circle-radius': 6,
})
```

**输入：**

- `res`：步骤 5 的结果。  
- `geolocusContext`：用于 `toGeoJSON` 和理解坐标系。  
- `map`：从 `useMapStore` 拿到的 Mapbox 实例。

**输出：**

- 地图上新增三个效果：  
  - 模糊区域 Polygon；  
  - 覆盖在那块区域上的概率 PNG 图层；  
  - 代表点（红色圆点）。
